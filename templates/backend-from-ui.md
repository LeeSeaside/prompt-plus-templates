# 后端接口需求文档生成提示词生成器

## 任务
请分析当前前端项目的页面结构和组件规范，生成一份用于"根据前端页面生成后端接口需求文档"的提示词。

---

## 分析步骤

### 1. 项目结构分析
- 页面文件存放位置
- 组件文件存放位置
- 路由配置方式
- 状态管理方式

### 2. 页面组织规范
- 页面目录结构
- 列表页面通用结构
- 表单页面通用结构
- 详情页面通用结构

### 3. 组件使用规范
- 表格组件（列定义方式）
- 表单组件（字段定义方式）
- 弹窗组件使用方式
- 分页组件使用方式

### 4. 数据交互模式
- 列表查询参数结构
- 表单提交数据结构
- 分页数据结构
- 下拉选项数据来源

---

## 输出要求

请根据以上分析，生成一份完整的提示词，格式如下：

```markdown
# [项目名] 后端接口需求文档生成规范

## 项目信息
- 前端框架：[Vue/React/Angular]
- UI组件库：[Element/Ant Design/其他]
- 页面目录：[路径]

## 页面分析规则

### 列表页面分析
- 表格列 → 字段定义
- 搜索表单 → 查询参数
- 操作按钮 → 接口需求

### 表单页面分析
- 表单字段 → 数据模型
- 校验规则 → 后端校验
- 下拉选项 → 字典接口

## 文档输出格式

生成的接口需求文档应包含：
1. 数据模型设计（表结构）
2. 接口列表（路径、方法、参数、响应）
3. 业务规则说明
4. 字典数据需求

## 使用说明

当需要为某个功能模块生成后端接口需求文档时，请提供：
1. 功能模块名称
2. 相关页面文件路径

我将分析页面代码，生成接口需求文档保存到 `docs/api-spec/[模块名].md`
```
